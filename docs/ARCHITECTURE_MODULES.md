# Архитектура tg_mini_app: модули и взаимодействия

Документ описывает **логическую структуру** репозитория: кто с кем связан, что передаёт и что получает. Условные блоки нарисованы **символами** (рамки из `+─│` или `┌┐└┘`); подписи **рядом** с блоком уточняют роль и обмен данными.

Область кода: каталог **`src/tg_mini_app/`** (пакет **`tg_mini_app`**).

---

## 1. Общая схема уровней

Система делится на **входы извне**, **два долгоживущих процесса** (HTTP API и бот), **общую модель данных** и **файлы витрины**.

```
                    ┌─────────────────────────────────────────────┐
  Браузер /         │  Telegram (клиенты: Mini App, чат бота)     │
  WebView           └─────────────────────┬───────────────────────┘
                                          │
         ┌────────────────────────────────┼────────────────────────────────┐
         │ HTTP(S)                        │ Bot API                        │
         ▼                                ▼                                │
    ┌─────────┐                     ┌─────────┐                            │
    │ FastAPI │                     │ aiogram │                            │
    │  app    │                     │   Bot   │                            │
    └────┬────┘                     └────┬────┘                            │
         │                               │                                 │
         │     общая бизнес-логика       │                                 │
         └───────────────┬───────────-───┘                                 │
                         ▼                                                 │
                  ┌─────────────┐                                          │
                  │ order_flow  │  статусы, проверки переходов             │
                  │ order_meta  │  позиции в meta, суммы                   │
                  └──────┬──────┘                                          │
                         │                                                 │
                         ▼                                                 │
                  ┌─────────────┐         ┌──────────────┐                 │
                  │ SQLAlchemy  │ ◄─────► │ SQLite/БД    │                 │
                  │   models    │         │ (файл БД)    │                 │
                  └─────────────┘         └──────────────┘                 │
                                                                           │
  Статика:  static/*.js, styles.css  ◄── отдаются FastAPI как /static/     │
  Шаблоны:  templates/*.html         ◄── /webapp, панели                   │
```

**Сверху (внешний мир):** пользователи и Telegram.

**Слева/справа условно:** HTTP приходит в FastAPI; события чата — в aiogram.

**Снизу:** персистентность через ORM и файл БД (URL из **`DATABASE_URL`**).

## 2. Точка входа HTTP: `api/app.py`

```
                      ┌──────────────────────────┐
  GET/POST …          │      api / app.py        │
  от клиента  ──────► │   create_app()           │
                      │   Uvicorn (процесс)      │
                      └────────────┬─────────────┘
           ┌───────────────────────┼───────────────────────┐
           │                       │                       │
           ▼                       ▼                       ▼
    ┌─────────────┐        ┌─────────────┐        ┌─────────────┐
    │ Подключает  │        │ startup:    │        │ Отдаёт      │
    │ роутеры     │        │ create_all, │        │ /static,    │
    │ (раздел 3)  │        │ seed, bot   │        │ Jinja2      │
    └─────────────┘        └─────────────┘        └─────────────┘
```

| Направление | Что происходит |
|-------------|----------------|
| **Получает** | HTTP-запросы: `/cart`, `/orders`, `/operator-panel`, `/catalog/*`, `/delivery/*`, `/webapp`, `/health`, … |
| **Отдаёт** | JSON (API), HTML (шаблоны), файлы CSS/JS |
| **Создаёт при старте** | `app.state.engine`, `session_factory`, при наличии токена — `app.state.bot` (**Aiogram Bot**) для уведомлений из API |
| **Вызывает** | `db.session.create_engine`, `schema_upgrade`, `seed_if_empty`, `include_router` |

## 3. Маршрутизаторы API (роутеры)

Каждый блок — **FastAPI APIRouter**; **внутрь** через dependency почти везде приходит **`AsyncSession`** (`api/deps.py` ← `request.app.state.session_factory`).

### 3.1. `api/cart.py` — корзина

```
        ┌────────────┐
  JSON  │  cart.py   │  JSON
  ────► │ /cart …    │ ────► ответ: состав, сумма, shared_key
        └─────┬──────┘
              │
              ▼
        ┌─────────────────────────────────────────┐
        │ models: Cart, CartItem, Product         │
        │ customer_identity (кто меняет корзину)  │
        └─────────────────────────────────────────┘
```

| Получает | Отправляет | Взаимодействует с |
|----------|------------|-------------------|
| `CartCreateRequest`, `CartItemChangeRequest`, заголовок **X-Telegram-Init-Data** или `customer_tg_id` | `CartResponse`, `ShareResponse` | **`customer_identity`**, **`models`**, БД |

---

### 3.2. `api/orders.py` — заказы клиента

```
        ┌─────────────┐
        │ orders.py   │
        │ /orders     │
        └──────┬──────┘
               │
   ┌───────────┼───────────┐
   ▼           ▼           ▼
┌─────────┐ ┌─────────┐ ┌──────────────────┐
│ Заказ   │ │ Корзина │ │ Уведомление      │
│ Order   │ │ lock    │ │ оператору (Bot)  │
│ meta    │ │         │ │ из app.state.bot │
└─────────┘ └─────────┘ └──────────────────┘
```

| Получает | Отправляет | Логика |
|----------|------------|--------|
| Создание заказа: `cart_id`, адрес, время, **initData** / `customer_tg_id` | `OrderResponse` (статус, позиции, маршрут оплаты и т.д.) | **`order_flow`** (статусы, допуски), **`order_meta`** (нормализация строк заказа), **`telegram_keyboards`** (клавиатуры в сообщениях) |
| Подпути: `/{id}/cancel`, `/{id}/substitutions/accept`, `/{id}/substitutions/reject` | Обновлённый заказ | Те же + запись в **`order.meta`** |

---

### 3.3. `api/operator_panel.py` — веб-панель оператора

```
     Браузер + cookie/Basic
            │
            ▼
     ┌──────────────┐
     │ operator_    │──────► HTML (Jinja2 templates)
     │ panel.py     │──────► POST: смена статуса, подстановки, маршрут
     └──────┬───────┘
            │
            ▼
     ┌──────────────────────────────────────┐
     │ Order, models, order_flow, order_meta│
     │ Иногда → Bot (клиенту / клавиатуры)  │
     └──────────────────────────────────────┘
```

| Получает | Отправляет | Прочее |
|----------|------------|--------|
| Формы, query-параметры фильтров | HTML-страницы, редиректы | **`require_operator_panel_auth`**, секрет **`OPERATOR_PANEL_TOKEN`** |

---

### 3.4. `api/catalog_admin.py` — JSON CRUD каталога (панель оператора)

Префикс роутера: **`/operator-panel/catalog`**. Тот же контроль доступа, что у **`operator_panel`** (**`require_operator_panel_auth`**: cookie после **`/operator-panel/login`** или HTTP Basic **`operator`** + **`OPERATOR_PANEL_TOKEN`**).

```
     ┌───────────────┐
     │ catalog_admin │  GET/POST/PATCH/DELETE
     │ /operator-    │  …/categories, …/products (JSON)
     │ panel/catalog │
     └───────┬───────┘
             │
             ▼
     ┌───────────────────────────────┐
     │ Category, Product,            │
     │ ProductAttribute, ProductImage│
     └───────────────────────────────┘
```

| Получает | Отправляет |
|----------|------------|
| JSON тела из **`schemas`** (категории, товары, атрибуты, URL фото) | JSON представлений (через **`catalog_serialize.product_to_dict`**) |

---

### 3.5. `api/catalog_panel.py` — HTML-редактор каталога и загрузка файлов

Префикс: **`/operator-panel/catalog-manage`**. Формы (Jinja2 **`catalog_manage.html`**, **`catalog_product_form.html`**), загрузка картинок через **`catalog_uploads.save_catalog_image`** в каталог **`catalog-media`** (смонтирован в **`app.py`** как **`StaticFiles`**).

| Получает | Отправляет |
|----------|------------|
| `multipart/form-data`, редиректы после POST | HTML или **303 Redirect** на список с `?ok=` / `?err=` |

Дублирования бизнес-логики с **`catalog_admin`** нет полного — веб-форма опирается на те же **`models`** и похожие операции с БД, но маршруты и формат ответа другие (HTML вместо JSON).

---

### 3.6. Витрина в `app.py` (не отдельный файл)

```
     GET /catalog/categories
     GET /catalog/products   ──►  list + join + selectinload
                                  ──► product_to_dict()
```

Публичный каталог **без** пароля; управление ассортиментом — в **`catalog_admin`** и **`catalog_panel`**.

### 3.7. `api/delivery_staff.py` — панель и API курьера

```
  COURIER_API_TOKEN (пусто → 404 на /delivery)
            │
            ▼
     ┌──────────────┐
     │ delivery_    │──────► HTML /delivery/login, cookie сессии
     │ staff.py     │──────► JSON API ?token=… (список заказов, delivered, cash-received)
     └──────────────┘
             │
             ▼
     order_flow (out_for_delivery → delivered), order_meta (курьерские поля)
```

| Получает | Отправляет |
|----------|------------|
| Секрет в query или cookie после **`/delivery/login`** | JSON или HTML; **401/403** при неверном токене |

## 4. Доменная логика заказов (общая для API и бота)

```
        ┌─────────────────┐
        │  order_flow.py  │
        │ константы       │
        │ OrderStatus     │
        │ require_*       │
        └────────┬────────┘
                 │
     проверки «можно ли из статуса A в B»
                 │
        ┌────────┴────────┐
        │   orders.py     │
        │ operator_       │
        │ panel.py        │
        │ delivery_       │
        │ bot/__main__    │
        └─────────────── -┘


        ┌─────────────────┐
        │  order_meta.py  │
        │ items в JSON    │
        │ total, статусы  │
        │ строк (замены)  │
        └────────┬────────┘
                 │
        ┌────────┴──────────────────────┐
        │ orders, operator_panel, bot   │
        └───────────────────────────────┘
```

| Модуль | Назначение | Кто читает/пишет |
|--------|------------|------------------|
| **`order_flow`** | Допустимость смены статуса, инварианты жизненного цикла | HTTP-роутеры, **`bot/__main__.py`** |
| **`order_meta`** | Структура **`order.meta["items"]`**, пересчёт суммы, поля курьера в meta | **`orders`**, **`operator_panel`**, частично отображение в Mini App |

## 5. Слой базы данных

```
                    ┌─────────────────┐
                    │  settings.py    │  DATABASE_URL, токены, BASE_URL
                    └────────┬────────┘
                             │
       ┌─────────────────────┼─────────────────────┐
       │                     │                     │
       ▼                     ▼                     ▼
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│ db/session  │      │ db/base.py  │      │ paths.py    │
│ create_eng… │      │ DeclarativeBase    │ пути к      │
│ FK SQLite   │      │             │      │ static, db  │
└──────┬──────┘      └──────┬──────┘      └─────────────┘
       │                    │
       │                    ▼
       │             ┌─────────────┐
       └────────────►│ db/models   │  ORM: Category, Product, Cart,
                     │             │  Order, ProductAttribute…
                     └──────┬──────┘
                            │
                     ┌──────┴──────┐
                     ▼             ▼
              ┌────────────┐ ┌────────────┐
              │ seed.py    │ │ schema_    │
              │ стартовые  │ │ upgrade.py │
              │ категории  │ │ новые      │
              │            │ │ колонки    │
              └────────────┘ └────────────┘
```

| Компонент | Получает | Отдаёт |
|-----------|----------|--------|
| **`session.create_engine`** | URL БД из **`get_settings()`** | **`AsyncEngine`** (отдельный экземпляр у **FastAPI** и у **`bot.Db`**) |
| **`models`** | — | Таблицы и связи для сессий SQLAlchemy |
| **`seed_if_empty`** | Пустая БД на старте API | Демо-категории и товары (один раз) |
| **`schema_upgrade`** | Существующая БД | `ALTER` недостающих колонок (напр. `stock_quantity`) |

**Важно:** процесс **Uvicorn** и процесс **бота** используют **разные** пулы соединений (каждый вызывает **`create_engine()`**), но один и тот же **файл БД** на диске.

## 6. Бот: `bot/__main__.py` и `bot/db.py`

```
  Telegram ──► updates ──► ┌──────────────┐
                           │ bot/__main__ │
                           │ Dispatcher   │
                           └──────┬───────┘
                                  │
          ┌───────────────────────┼───────────────────────┐
          ▼                       ▼                       ▼
   ┌────────────┐         ┌────────────┐         ┌────────────┐
   │ bot/db.Db  │         │ order_flow │         │ models +   │
   │ session()  │         │ order_meta │         │ select/    │
   │            │         │            │         │ commit     │
   └────────────┘         └────────────┘         └────────────┘
                                  │
                                  ▼
                           ┌────────────┐
                           │ Bot API    │
                           │ ответы     │──► пользователю / оператору
                           │ клавиатуры│     (telegram_keyboards)
                           └────────────┘
```

| Получает | Отправляет |
|----------|------------|
| Команды, callback, сообщения из Telegram | Сообщения, кнопки, **`MenuButtonWebApp`**, напоминания по заказам |
| Доступ к **`Order`**, **`Cart`** через **`bot.db`** | Обновления статусов и **`meta`** (согласовано с HTTP-частью) |

**`bot/db.py`:** обёртка **`create_engine` + sessionmaker** только для бота (не разделяет объект `app.state` с FastAPI).

## 7. Презентация и статика

```
  ┌─────────────┐
  │ templates/  │  base.html, webapp.html, operator_*, …
  └──────┬──────┘
         │ Jinja2
         ▼
  ┌─────────────┐      ┌─────────────┐
  │ app.py      │      │ static/     │
  │ TemplateResp│      │ app_v3.js   │
  │             │      │ styles.css  │
  └─────────────┘      └──────┬──────┘
                              │
  Браузер ◄───────────────────┘  GET /static/…
```

| Файл | Роль |
|------|------|
| **`app_v3.js`** | Витрина: `/catalog/*`, `/cart`, `/orders`, опрос статуса заказа |
| **`styles.css`** | Оформление Mini App и общие кнопки |
| **`webapp.html`** | Разметка корзины, баннера заказа, вкладок категорий |

**Получает с API:** JSON. **Отправляет:** JSON + заголовок **X-Telegram-Init-Data** (если есть).

## 10. Запуск процессов (не код, но полезно для карты)

| Процесс | Точка входа (типично) | Назначение |
|---------|------------------------|------------|
| HTTP API | **`python -m tg_mini_app.api`** или Uvicorn на **`create_app()`** | Всё, что в §2–§5 с запросами |
| Бот | **`python -m tg_mini_app.bot`** | Опрос Telegram (long polling), сцены §6 |

Оба должны видеть один **`.env`** / переменные окружения (**`settings`** читает **`paths.ENV_FILE`**).
